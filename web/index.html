About this project

Simple guide

What it does

CyberSynth IDS lets you test network-flow CSV files with a trained intrusion detection model and create synthetic rows for experiments.

Who it is for

This project is useful for students and learners who want a local, Docker-ready tool for dataset testing, model scoring, and controlled synthetic data experiments.

What to check

For uploaded data, check feature coverage before scoring. For generated data, check the summary, exact-match result, and model results before using it.

Current limit

The generated data is best used for testing and augmentation. It should not replace real network traffic when measuring final model performance.

How to use

Recommended workflow
  1. Open OverviewCheck that the model, reports, and required files are ready.
  2. Use DatasetUpload a CSV, validate its columns, then score it with the active trained model.
  3. Create DataChoose traffic type, attack family, source, and row count, then generate a dataset.
  4. Open ResultsReview model performance, category detection, dataset mix, and report downloads.
  5. Use Advanced ToolsPredict one flow, explain a result, or check drift when you need deeper inspection.

Different datasets

How the model stays consistent

Shared schema

Preprocessing maps each raw dataset into the same flow columns, labels, categories, source names, and environment tags before training.

Source-safe training

The model learns from traffic features. Source, environment, and category metadata are kept for reporting but removed from model inputs.

Unknown values

New services or protocols in an uploaded CSV are handled without breaking prediction, but the Drift page should be used before trusting the scores.

Retraining rule

Add new datasets through preprocessing, retrain the model, then check per-category metrics and drift before using the model in a report.
